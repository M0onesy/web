# 智龄协同平台展示站部署手册

这份文档面向“低成本、免备案、尽快上线”的目标，结合当前项目的实际结构，给出两套最适合的上线方案：

1. 首选：`EdgeOne Pages`
2. 次选：`腾讯云 COS 中国香港静态网站托管`

这两套方案都适合当前这个 `Vue + Vite` 静态站。

## 一、先确认项目参数

当前项目的部署信息已经明确：

- 项目目录：`C:\Users\20719\Desktop\三创\web`
- 构建命令：`npm run build`
- 发布目录：`dist`
- 路由模式：`Hash` 路由

补充说明：

- 由于你当前使用的是 `Hash` 路由，因此静态托管时通常不需要额外配置单页应用回退规则。
- 只要 `dist` 目录能正常构建出来，这个网站就可以直接上线。

## 二、上线前准备

无论你选哪种方案，建议先完成下面这几步。

### 1. 安装依赖并本地确认可运行

```bash
cd C:\Users\20719\Desktop\三创\web
npm install
npm run dev
```

浏览器打开：

```text
http://127.0.0.1:5173
```

确认首页和几个子页面都能正常访问。

### 2. 生成正式发布文件

```bash
cd C:\Users\20719\Desktop\三创\web
npm run build
```

成功后会得到：

```text
C:\Users\20719\Desktop\三创\web\dist
```

这个 `dist` 文件夹就是部署时真正要发布的内容。

### 3. 上线前建议检查的内容

- 首页能否正常打开
- 各导航页面能否切换
- 图片和图标是否正常显示
- 浏览器标签图标是否正常
- 手机端打开时布局是否正常

## 三、首选方案：EdgeOne Pages

这条路线最适合当前你的情况：

- 需要尽快上线
- 不想备案
- 访问者较多来自中国大陆
- 不希望继续用 `Cloudflare Pages`

### 方案特点

优点：

- 对中国大陆访问通常比 `Cloudflare Pages` 更友好
- 支持静态网站和常见前端框架
- 支持持续部署
- 适合展示站

限制：

- 如果你的域名没有 ICP 备案，自定义域名通常只能走“全球可用区（不含中国大陆）”
- 这意味着它仍不等于真正的大陆备案站，只是通常体验会比 `Cloudflare Pages` 更好

### A. 最快上线方式：本地构建后直接部署

#### 第一步：安装 EdgeOne CLI

在 PowerShell 中执行：

```bash
npm install -g edgeone
```

安装完成后验证：

```bash
edgeone -v
```

#### 第二步：登录 EdgeOne

```bash
edgeone login
```

登录后如果提示站点或区域，按控制台提示完成。

#### 第三步：进入项目目录

```bash
cd C:\Users\20719\Desktop\三创\web
```

#### 第四步：构建项目

```bash
npm run build
```

#### 第五步：部署项目

可以先用一个英文项目名，例如：

```bash
edgeone pages deploy -n zhiling-collab
```

说明：

- `zhiling-collab` 是你在平台上的项目名，可以换成别的英文名
- 对你当前这个项目，直接在 `web` 目录下执行即可
- 平台会根据当前项目结构进行部署

#### 第六步：查看返回地址

部署成功后，CLI 会返回一个访问链接。先打开这个链接，确认：

- 首页能打开
- 菜单能跳转
- 子页面能打开
- 样式和图片正常

### B. 更适合长期维护的方式：连接 Git 自动部署

如果后续你还要继续改站点，更推荐这个方式。

#### 第一步：把代码推到 GitHub

建议把 `web` 目录上传到 GitHub 仓库。

#### 第二步：在 EdgeOne Pages 中导入 Git 仓库

登录控制台后：

1. 创建项目
2. 选择从 Git 导入
3. 授权 GitHub
4. 选择你的仓库

#### 第三步：填写构建参数

按下面填写：

- Build command：`npm run build`
- Output directory：`dist`
- Root directory：
  - 如果仓库根目录就是 `web`，留空
  - 如果仓库根目录是 `三创`，填写 `web`

#### 第四步：触发首次部署

创建完成后，平台会自动构建和发布。

以后每次提交代码到主分支，平台都会自动重新部署。

### C. 绑定自定义域名

#### 第一步：准备域名

推荐优先使用子域名，例如：

- `www.你的域名.com`
- `demo.你的域名.com`

这样会比直接绑根域名更省事。

#### 第二步：添加自定义域名

进入 EdgeOne Pages 项目设置，找到自定义域名，添加你的域名。

#### 第三步：配置 DNS

平台会提供一个目标地址。到你的域名解析平台中添加：

- 记录类型：`CNAME`
- 主机记录：`www` 或 `demo`
- 记录值：平台提供的目标地址

#### 第四步：等待 HTTPS 生效

等待 DNS 生效后，平台通常会自动处理 HTTPS 证书。证书生效后，再用 `https://` 访问。

### D. 后续如何更新

如果你使用 CLI 手动部署，更新流程就是：

```bash
cd C:\Users\20719\Desktop\三创\web
npm run build
edgeone pages deploy -n zhiling-collab
```

如果你使用 Git 自动部署，则只需要：

1. 改代码
2. 提交到仓库
3. 平台自动重新发布

### E. 常见问题

#### 1. 需要额外配置路由回退吗

一般不需要。

原因是你当前项目使用的是 `Hash` 路由，URL 中的页面切换主要发生在 `#` 之后，不依赖服务器识别子路径。

#### 2. 没备案能不能用

可以，但通常不能享受中国大陆正式加速区。

#### 3. 为什么它比 Cloudflare 可能更快

本质上是平台链路和节点策略不同。它并不是把“免备案网站”变成“大陆备案站”，而是通常对大陆访问更友好一些。

## 四、次选方案：腾讯云 COS 中国香港静态网站托管

这套方案适合你想要：

- 保持免备案
- 自己完全掌控静态文件
- 结构尽量简单
- 想要一个更容易理解的“上传文件即上线”方案

### 方案特点

优点：

- 逻辑简单
- 文件托管透明
- 香港节点对中国大陆通常比部分海外平台更稳
- 非常适合静态展示站

限制：

- 默认域名不适合正式公开使用
- 正式上线最好绑定你自己的域名
- 如果后续叠加中国大陆 CDN，则通常会涉及备案问题

### A. 创建 COS 存储桶

#### 第一步：登录腾讯云 COS 控制台

打开：

- [腾讯云 COS 控制台](https://console.cloud.tencent.com/cos5)

#### 第二步：创建存储桶

创建时建议这样填写：

- 地域：`中国香港`
- 访问权限：`公有读私有写`

说明：

- 纯静态展示站优先这样设置，最省事
- 如果设置得过于严格，初期很容易直接出现 `403`

### B. 开启静态网站功能

进入存储桶后：

1. 进入 `基础配置`
2. 找到 `静态网站`
3. 开启静态网站
4. 索引文档填写：`index.html`

由于你当前是 `Hash` 路由，因此只配 `index.html` 通常就够了。

### C. 构建网站文件

本地执行：

```bash
cd C:\Users\20719\Desktop\三创\web
npm run build
```

### D. 上传 `dist` 内容到存储桶

#### 正确上传方式

要上传的是 `dist` 目录中的内容，而不是整个 `dist` 文件夹本身。

正确结果应该是：

```text
/index.html
/assets/...
/brand/...
```

不要传成：

```text
/dist/index.html
```

#### 上传步骤

1. 进入 COS 存储桶
2. 打开 `文件列表`
3. 选择本地 `dist` 目录中的所有文件和文件夹
4. 上传到存储桶根目录

### E. 绑定自定义域名

正式上线建议一定绑定自己的域名。

#### 第一步：添加自定义源站域名

在存储桶内进入：

- `域名与传输管理`
- `自定义源站域名`
- 点击添加域名

建议填写：

- 域名：`www.你的域名.com`
- 源站类型：`静态网站源站`

#### 第二步：配置 DNS

COS 会提供一个 CNAME 地址，到你的域名解析后台中添加：

- 记录类型：`CNAME`
- 主机记录：`www`
- 记录值：COS 提供的目标地址

### F. 配置 HTTPS 证书

这是非常关键的一步，否则正式访问体验会差很多。

#### 第一步：申请证书

如果你还没有 SSL 证书，可以在腾讯云 SSL 控制台申请免费证书。

#### 第二步：绑定证书

回到 COS：

- `域名与传输管理`
- `自定义源站域名`
- 找到你的域名
- 绑定 HTTPS 证书

#### 第三步：等待生效

一般等 DNS 和证书同步完成后，用 `https://www.你的域名.com` 访问。

### G. 后续更新方式

每次改完网站后，重复两步：

```bash
cd C:\Users\20719\Desktop\三创\web
npm run build
```

然后把新的 `dist` 内容重新上传覆盖到 COS 存储桶即可。

### H. 常见问题

#### 1. 为什么访问时浏览器下载文件而不是打开网页

这是腾讯云近年来对默认访问域名策略调整后的常见现象。正式上线时不要依赖默认域名，应该绑定自己的自定义域名。

#### 2. 为什么会报 403

常见原因：

- 存储桶权限不对
- 文件没有传到根目录
- 自定义域名未生效
- 证书或域名配置还没完成

#### 3. 能不能再套腾讯云 CDN

技术上可以，但如果要对中国大陆做正式加速，通常就会回到备案问题。

## 五、如何在首选和次选之间做选择

### 选 EdgeOne Pages 的情况

适合你如果：

- 想要更接近“平台托管”体验
- 希望后续接 Git 自动发布
- 想要比 Cloudflare 更适合大陆访问
- 想要更少自己维护静态文件

### 选 COS 中国香港 的情况

适合你如果：

- 想完全掌控静态文件结构
- 更喜欢“生成 dist 然后上传文件”的方式
- 想做一个稳定的香港静态托管备份入口

### 实际建议

对于你现在这个项目，建议采用下面顺序：

1. 先上 `EdgeOne Pages`
2. 如果你还想保留一个备用入口，再补一份 `COS 中国香港` 部署
3. 然后从手机网络、校园网、家宽分别实测访问速度

## 九、最后提醒

如果你的目标以后从“能打开”升级为“全国访问都明显快而且稳”，那就需要重新考虑：

- ICP 备案
- 中国大陆可用区
- 国内 CDN 或边缘加速

也就是说：

- `免备案`
- `低成本`
- `中国大陆极致速度`

这三者通常不能同时拉满。

当前阶段最平衡的选择，仍然是：

1. `EdgeOne Pages`
2. `COS 中国香港` 作为备选或补充
